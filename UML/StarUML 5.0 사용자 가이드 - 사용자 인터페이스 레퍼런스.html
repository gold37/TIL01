메인 윈도우

StarUML™의 메인 윈도우는 다음과 같이 구성되어 있습니다.

메인 메뉴

화면 최상단에 메인 메뉴가 위치하고 있습니다. StarUML™의 거의 대부분의 기능들은 메인 메뉴에서 찾아서 실행할 수 있습니다.

도구 모음

도구 모음은 메인 메뉴 바로 아래에 위치하고 있으며 사용자가 자주 많이 사용하는 메뉴들로 구성되어 있습니다.

브라우저 영역

브라우저 영역은 왼쪽 상단에 위치하고 있으며 소프트웨어 프로젝트의 구성 요소들을 쉽게 탐색할 수 있도록 도와주는 기능들을 포함하고 있습니다. 여기에는 모델 요소들을 계층적으로 보여주는 모델 탐색기와 다이어그램들을 분류별로 보여주는 다이어그램 탐색기가 있습니다.

인스펙터 영역

인스펙터 영역은 왼쪽 하단에 위치하고 있으며 하나의 요소에 대한 세부적인 정보들을 편집하기 위한 기능들을 포함하고 있습니다. 여기에는 프로퍼티를 편집하기 위한 프로퍼티 편집기, 세부 설명을 기록하기 위한 문서화 편집기, 그리고 부가적인 파일이나 URL을 첨부하기 위한 첨부 파일 편집기가 있습니다.

정보 영역

정보 영역은 오른쪽 하단에 위치하고 있으며 StarUML™ 전반에 걸친 정보들을 보기 위한 기능들을 포함하고 있습니다. 로그 기록을 보여주는 출력 창, 모델 검색 혹은 검사 결과를 보여주는 메시지 창이 있습니다.

다이어그램 영역

다이어그램 영역은 오른쪽 상단에 위치하고 있으며 다이어그램을 실제 편집하고 열려진 다이어그램들을 관리하기 위한 기능들을 담고 있습니다.

팔레트

각 다이어그램별로 생성할 수 있는 요소들에 대한 팔레트가 분류되어 구성되어 있습니다.

표준 도구모음

표준 도구모음에는 다음과 같은 기능들을 제공하고 있습니다.

도구 설명
New Project[Ctrl+N] 새로운 프로젝트를 만듭니다. 
Open[Ctrl+O] 프로젝트 파일을 불러옵니다. 
Save[Ctrl+S] 프로젝트 파일을 저장합니다.
Print[Ctrl+P] 다이어그램을 프린트로 인쇄합니다. 
Cut[Ctrl+X] 선택된 요소들을 클립보드에 복사하고 잘라냅니다.
Copy[Ctrl+C] 선택된 요소들을 클립보드에 복사합니다.
Paste[Ctrl+V] 클립보드의 내용을 현재 선택된 요소(혹은 다이어그램)로 붙여 넣습니다.
Delete[Del] 다이어그램에 선택된 뷰 요소들을 삭제합니다.
Undo[Ctrl+Z] 사용자가 가장 최근에 수행한 액션을 취소합니다.
Redo[Ctrl+Y] 사용자가 취소한 액션을 다시 실행합니다.
Find[Ctrl+F] 요소를 검색합니다.
Collection Editor[Ctrl+F5] 현재 선택된 모델 요소의 하위 요소들을 편집하기 위한 컬렉션 편집기가 나타납니다.
Constraints[Ctrl+F6] 현재 선택된 모델 요소의 제약사항을 편집하기 위한 제약사항 편집기가 나타납니다.
TaggedValues[Ctrl+F7] 현재 선택된 모델 요소의 확장속성을 편집하기 위한 확장속성 편집기가 나타납니다.
Profiles[Ctrl+F8] 프로파일 관리자가 나타납니다.
Verify Model[F9] 현재 프로젝트에 존재하는 모델 요소들을 검사하기 위한 모델 검사 대화상자가 나타납니다.

Suppress Literals[Shift+Ctrl+L] 선택된 열거형 뷰의 속성을 표시하는 영역을 감추어 나타내지 않습니다.

Show Operation Signature 선택된 뷰 요소들(e.g. 클래스, 서브시스템 등)의 연산을 시그너쳐까지 완전하게 표현합니다.
Show Properties 선택한 뷰 요소들의 확장 속성 등의 프로퍼티를 표시합니다.
Show Compartment Visibility 선택된 뷰 요소들이 가진 구획(e.g. 속성 구획, 연산 구획 등) 항목들의 가시성을 표시합니다.
Show Compartment Stereotype 선택된 뷰 요소들이 가진 구획(e.g. 속성 구획, 연산 구획 등) 항목들의 스테레오타입을 텍스트로 표시합니다.

팔레트 도구모음

팔레트 도구모음은 다음과 같이 다이어그램에서 요소를 선택하거나  생성하기 위한 기능들을 제공하고 있습니다.

비 UML 요소 팔레트 도구 

UML 요소가 아니고 일반적인 용도로 사용되는 요소를 생성할 수 있는 기능들입니다. Annotation 팔레트에 포함되어 있으며, 모든 다이어그램에서 활성화됩니다.

도구 설명
Select 다이어그램에서 요소를 선택하거나 선택된 요소에 대해 이동,크기조정 등의 기능을 수행하는데 사용되는 가장 기본적인 도구입니다.
노트 Note 현재 다이어그램에 노트 요소를 생성합니다.
노트 연결 Note Link 현재 다이어그램에 노트를 다른 요소에 연결시킵니다.
텍스트 Text 현재 다이어그램에 문자열 요소를 생성합니다.
Rectangle 현재 다이어그램에 사각형 도형을 생성합니다. 
Ellipse 현재 다이어그램에 원형(타원형) 도형을 생성합니다.
Rounded Rectangle 현재 다이어그램에 모서리가 둥근 사각형을 생성합니다.

Swimlane(Vertical) 현재 다이어그램에 세로방향의 파티션(Swimlane) 요소를 생성합니다. 활동 다이어그램
Swimlane(Horizontal) 현재 다이어그램에 가로방향의 파티션(Swimlane) 요소를 생성합니다. 활동 다이어그램
Association 현재 다이어그램에서 두 클래스류사이에 의미적 연관(Association) 관계를 연결합니다. 클래스 다이어그램, 컴포넌트 다이어그램, 디플로이먼트 다이어그램, 유스케이스 다이어그램
DirectedAssociation 현재 다이어그램에서 두 클래스류사이에 방향을 가지는 연관(Directed Association) 관계를 연결합니다. 클래스 다이어그램, 디플로이먼트 다이어그램, 유스케이스 다이어그램
Aggregation 현재 다이어그램에서 두 클래스류사이에 집합 연관(Aggregation) 관계를 연결합니다. 클래스 다이어그램
Composition 현재 다이어그램에서 두 클래스류사이에 복합연관(Composition) 관계를 연결합니다. 클래스 다이어그램
Generalization 현재 다이어그램에서 더 일반적인 요소와 더 구체적인 요소를 일반화(Generalization) 관계로 연결합니다. 클래스 다이어그램, 유스케이스 다이어그램
Dependency 현재 다이어그램에서 두 요소 사이를 의존(Dependency)관계로 연결합니다. 클래스 다이어그램, 컴포넌트 다이어그램, 디플로이먼트 다이어그램, 유스케이스 다이어그램,복합구조 다이어그램
Realization 현재 다이어그램에서 명세요소와 그것을 구현하는 요소를 실체화(Realization) 관계로 연결합니다. 클래스 다이어그램, 컴포넌트 다이어그램, 복합구조 다이어그램
AssociationClass 현재 다이어그램에서 클래스(Class)와 연관(Association)을 연결하여 연관자체가 클래스의 의미를 가질 수 있도록 합니다. 클래스 다이어그램
Include 현재 다이어그램에서 한 유스케이스가 다른 유스케이스의 행위를 포함할 수 있도록 포함(Include) 관계로 연결합니다. 유스케이스 다이어그램
Extend 현재 다이어그램에서 한 유스케이스가 특정 유스케이스에 정의된 행위로 추가 확장될 수 있도록 확장(Extend)관계로 연결합니다. 유스케이스 다이어그램
AssociationRole 현재 다이어그램에서 두 역할 사이를 연관-역할(AssociationRole)로 연결합니다. 협동 역할 다이어그램
SelfAssociationRole 현재 다이어그램에서 한 역할에서 그 자체로 연관-역할(AssociationRole)을 생성합니다. 협동 역할 다이어그램
Link 현재 다이어그램에서 두 객체 사이를 링크(Link)로 연결합니다. 클래스 다이어그램, 협동 다이어그램
SelfLink 현재 다이어그램에서 한 객체에서 그 자체로 링크(Link)를 생성합니다. 클래스 다이어그램, 협동 다이어그램
ForwardMessage 현재 다이어그램에서 두 역할 사이의 메세지(Message)를 정의합니다. 시퀀스 역할 다이어그램, 협동 역할 다이어그램
ReverseMessage 현재 다이어그램에서 두 역할 사이의 메세지(Message)를 정의합니다. 시퀀스 역할 다이어그램, 협동 역할 다이어그램
SelfMessage 현재 다이어그램에서 한 역할에서 그 자체로 메세지(Message)를 생성합니다. 시퀀스 역할 다이어그램, 협동 역할 다이어그램
ForwardStimulus 현재 다이어그램에서 두 객체 사이의 자극(Stimulus)을 정의합니다. 시퀀스 다이어그램, 협동 다이어그램
ReverseStimulus 현재 다이어그램에서 두 객체 사이의 자극(Stimulus)을 정의합니다. 시퀀스 다이어그램, 협동 다이어그램
SelfStimulus 현재 다이어그램에서 한 객체에서 그 자체로 자극(Stimulus)을 생성합니다. 시퀀스 다이어그램, 협동 다이어그램
Transition 현재 다이어그램에서 원시 상태로부터 목표상태로 전이(Transition)을 연결합니다. 상태 다이어그램, 활동 다이어그램
SelfTransition 현재 다이어그램에서 한 상태에서 그 상태로 전이(Transition)을 연결합니다. 상태 다이어그램, 활동 다이어그램
Connector 현재 다이어그램에서 한 특성(Feature)에서 그 특성(Feature)로 연결자(Connector)을 연결합니다. 클래스 다이어그램, 컴포넌트 다이어그램, 디플로이먼트 다이어그램, 복합구조 다이어그램

윈도우

모델 탐색기 

모델 탐색기는 사용자가 모델 요소들을 계층적으로 보여줌으로써 효과적으로 관리하고 탐색할 수 있도록 지원합니다. 모델 탐색기는 [Browser] 영역에서 [Model Explorer] 탭을 선택하면 나타납니다.

다이어그램 탐색기 

다이어그램 탐색기는 사용자가 다이어그램들을 분류 별로 효과적으로 관리하고 탐색할 수 있도록 지원합니다. 다이어그램 탐색기는 [Browser] 영역에서 [Diagram Explorer] 탭을 선택하면 나타납니다.

프로퍼티 편집기 

프로퍼티 편집기는 현재 선택된 모델 요소의 세부적인 프로퍼티를 편집하기 위해서 사용됩니다. 프로퍼티 편집기는 [Inspector] 영역에서 [Properties] 탭을 선택하면 나타납니다.

문서화 편집기

문서화 편집기는 현재 선택된 요소에 부가적인 설명을 기록하기 위해서 사용됩니다. 문서화 편집기는 [Inspector] 영역에서 [Documentation] 탭을 선택하면 나타납니다.

첨부파일 편집기 

첨부파일 편집기는 사용자가 특정 요소에 파일 혹은 웹 주소(URL)를 첨부할 수 있도록 합니다. 첨부파일 편집기는 [Inspector] 영역에서 [Attachments] 탭을 선택하면 나타납니다.

첨부 목록

요소에 첨부된 파일 혹은 URL의 목록이 나타납니다.

열기 버튼

첨부 목록에서 선택된 첨부 파일 혹은 URL을 연결된 프로그램으로 열어줍니다. 예를 들어 .hwp 파일의 경우에는 자동으로 한글 워드프로세서 프로그램이 실행되고 http://www.staruml.com 과 같이 주소인 경우에는 웹 브라우저에서 열립니다.

추가 버튼 

새로운 파일 혹은 URL을 첨부합니다. 이 버튼을 누르면 첨부 항목 편집기가 나타납니다.

삭제 버튼

첨부 목록에서 선택된 항목을 삭제합니다.

편집 버튼

첨부 목록에서 선택된 항목을 편집합니다. 파일 이름을 고친다든지 혹은 다른 URL의 주소를 입력할 수 있도록 첨부 항목 편집기가 나타납니다.

위로 이동 버튼 

첨부 목록에서 선택된 항목을 위로 이동합니다.

아래로 이동 버튼

첨부 목록에서 선택된 항목을 아래로 이동합니다.

첨부 항목 대화상자

첨부 항목의 내용을 편집합니다. 이 부분에는 특정 URL 혹은 파일의 경로명을 입력해야 하며, 오른쪽 버튼을 눌러 파일을 직접 선택할 수 도 있습니다.

출력 창 

출력 창은 StarUML™에서 일어나는 사건들의 기록을 남겨서 보여주는 기능을 수행합니다. 출력 창은 [Information] 영역에서 [Output] 탭을 선택하면 나타납니다.

메시지 창

메시지 창은 요소를 검색한 결과 혹은 소프트웨어 모델의 검사 결과들을 나타내어 주는 기능을 합니다. 메시지 창은 [Information] 영역에서 [Message] 탭을 선택하면 나타납니다.

Dialogs

컬렉션 편집기

컬렉션 편집기는 특정 요소가 하위에 포함하고 있는 요소들의 리스트를 관리하기 위해 사용됩니다.

탭

요소가 포함하고 있는 컬렉션(현재 요소가 포함하고 있는 하위 요소들의 리스트)들을 탭으로 나타냅니다. 요소의 종류에 따라 가지는 컬렉션들이 다르므로 탭의 구성도 달라집니다. 예를 들어 클래스(Class) 요소의 경우에는 하위 요소로 속성(Attribute)과 연산(Operation)을 가지므로 해당 탭이 나타납니다. 관계(Relations) 탭은 요소에 관계 없이 항상 나타납니다.

컬렉션 요소 목록

하위 요소들의 목록이 나타납니다. 이 곳에서 하나의 요소를 선택하면서 인스펙터 영역의 프로퍼티 편집기, 문서화 편집기, 첨부 파일 편집기를 사용하여 편집할 수 있습니다. 이 목록에서 요소의 스테레오타입 혹은 가시성/스테레오타입 등을 나타나게 하는 등의 지정을 하려면 환경 설정하기의 일반 설정을 참고하세요.

추가 버튼

새로운 요소를 생성해서 목록에 추가합니다. 경우에 따라서는 새로운 요소를 생성하지 않고 기존의 요소를 연결하는 용도가 되는 경우(e.g. Residents, DeployedComponents, RaisedSignals)도 있습니다.

삭제 버튼

컬렉션 요소 목록에서 선택된 요소를 삭제합니다. 경우에 따라서는 요소를 삭제하지 않고 리스트에서 제거하는 용도가 되는 경우(e.g. Residents, DeployedComponents, RaisedSignals)도 있습니다.

위로 이동 버튼

컬렉션 요소 목록에서 선택된 요소를 위로 이동합니다.

아래로 이동 버튼

컬렉션 요소 목록에서 선택된 요소를 아래로 이동합니다.

제약사항 편집기

제약사항 편집기는 요소에 부여된 제약사항(Constraint)들을 관리하게 위해 사용됩니다.

Constraints

제약사항 목록에는 요소에 부여된 제약(Constraint)들의 이름과 내용을 리스트로 나타냅니다.

Add

요소에 새로운 제약을 부여합니다. 이 버튼을 누르면 제약 항목 대화상자가 나타납니다.

Delete

제약사항 목록에 선택된 제약을 삭제합니다.

Edit

제약사항 목록에 현재 선택된 제약의 내용을 변경합니다.

Move Up

제약사항 목록에 현재 선택된 제약을 위로 이동합니다.

Move Down

제약사항 목록에 현재 선택된 제약을 아래로 이동합니다.

제약사항 대화상자 

제약사항 편집기에서 하나의 새로운 제약을 추가하거나 기존의 제약의 이름 혹은 내용을 편집하고자 할 때 사용됩니다. [Name]에는 제약사항의 이름을 입력하고 [Body] 부분에는 제약의 실제 내용을 입력합니다. 이 부분에는 사용자가 자유롭게 내용을 입력할 수도 있고 UML의 OCL(Object Constraint Language) 언어로 입력할 수 도 있습니다.

확장속성 편집기

확장속성 편집기는 특정 요소에 추가적으로 부여할 수 있는 확장속성(TaggedValue)들의 값을 편집하기 위해 사용됩니다.

프로파일 탭

기본적으로 확장 속성은 프로파일에서 정의됩니다. 현재 선택된 요소에 적용될 수 있는 확장 속성을 가진 프로파일이 있다면 탭 부분에 나타나며, 해당 프로파일에서 정의된 확장속성들이 [Tag Definition Set]과 [Tagged Values] 부분에 나타납니다.

확장 속성 집합

현재 선택된 요소에 적용될 수 있는 확장속성-집합(TagDefinitionSet)들이 나타납니다. 이 집합에 포함된 확장속성들은 [Tagged Values] 부분에 나타납니다.

꼬리 값

[Tag Definition Set] 부분에 선택된 확장속성-집합에 포함된 속성들이 열거되고 그것들의 값이 표현됩니다. 그리고 사용자는 직접 이 값들을 변경할 수 있습니다.

기본값 지정

확장속성에는 항상 기본값(Default Value)가 존재합니다. [Tagged Values] 부분에 하나의 속성을 선택하고 이 버튼을 누르면 변경되었던 값들을 지우고 기본값으로 설정합니다.

프로파일 관리자

프로파일 관리자는 현재 프로젝트에서 사용할 UML 프로파일들을 포함시키거나 제외시킬 수 있습니다.

사용가능한 프로파일들

StarUML™에서 사용할 수 있도록 등록된 UML 프로파일들의 목록을 나타냅니다. 단, 현재 프로젝트에서 사용 중인 프로파일들은 이 부분에 나타나지 않습니다.

사용중인 프로파일들

현재 프로젝트에서 사용 중인 UML 프로파일들의 목록을 나타냅니다.

큰아이콘/작은아이콘 버튼 

프로파일 목록에 나타나는 항목들의 아이콘을 크게 하거나 작게 나타날 수 있습니다. 만약 프로파일의 이름이 모두 보이지 않아 불편하다면 작은 아이콘으로 나타나도록 버튼을 선택합니다.

포함하기

사용 가능한 프로파일에서 선택된 항목을 현재 프로젝트에서 사용할 수 있도록 포함합니다.

제외하기

포함된 프로파일에서 선택된 항목을 현재 프로젝트에서 사용할 수 없도록 제외합니다.

Add-In 관리자

Add-In 관리자는 어떤 Add-In들이 설치되어 있는지 그리고 그것을 실제 사용하도록 활성화 할 것인지 아니면 사용하지 못하도록 비 활성화할 것인지를 사용자가 체크할 수 있습니다.

Add-Ins 목록

설치된 Add-In 들의 목록이 나타납니다. 사용자는 각 항목을 체크하거나 체크를 해제함으로써 해당 Add-In을 사용 가능 혹은 사용 불가능하게 할 수 있습니다.

  • Add-In 관리자 화면에 나타나는 Add-In 목록은 사용자의 설치 환경에 따라 다를 수 있습니다.

새 프로젝트 선택 대화상자

새 프로젝트 선택 대화상자는 사용자가 새로운 프로젝트를 생성할 때 선택할 수 있는 다양한 방법을 제공합니다. 새 프로젝트 대화상자는 Approaches, Open Files, 그리고 Recent Files 세 개의 페이지로 구성되어있습니다.

Approaches

사용자가 새로운 프로젝트를 생성할 때 특정 접근법(Approach)을 적용할 수 있도록 해 줍니다.

  • Approaches: 접근법 목록에는 등록된 접근법들의 아이콘과 함께 나타납니다. "빈 프로젝트" 항목은 어떤 접근법도 적용하고 싶지 않을 때 선택합니다.
  • 큰아이콘/작은아이콘 버튼: 접근법 목록에 나타나는 항목들의 아이콘을 크게 하거나 작게 나타낼 수 있습니다. 만약 접근법의 이름이 모두 보이지 않아 불편하다면 작은 아이콘으로 나타나도록 버튼을 선택합니다. 
  • Description: 접근법 목록에 사용자가 항목을 선택하면 해당 접근법에 대한 간략한 설명이 나타나는 영역입니다. 
  • Set As Default Approach 체크 박스: 접근법 목록에서 사용자가 기본 접근법으로 사용고자 하는 접근법을 선택한 후 이 체크 박스를 클릭하면 해당 접근법이 기본 접근법으로 설정됩니다. 기본 접근법은 새 프로젝트([File]->[New Project])를 선택했을 때 생성되는 프로젝트에 적용되는 접근법입니다.

파일 열기

기존에 만들어 진 파일의 열기 기능을 제공합니다. 왼쪽의 트리뷰는 사용자 시스템의 폴더 구조를 보여주며, 오른쪽 파일 목록 부분은 선택된 폴더에 있는 프로젝트 파일들의 목록을 보여줍니다. 이 파일 목록에서 열고자 하는 파일을 선택한 후 [Open] 버튼을 선택하면 선택된 파일이 열립니다.

최근 파일들

최근에 편집했던 파일들의 목록을 보여주고 새로 열 수 있는 기능을 제공합니다.

  • Recent Files: 최근에 편집했던 파일들의 목록을 보여줍니다.
  • Delete not existing files from the list 버튼: 최근에 편집한 파일 목록에 나타난 파일들 중에 더 이상 존재하지 않는 파일들을 검사하여 최근 파일 목록에서 삭제합니다.
  • Delete all recent files in the list 버튼: 최근에 편집한 파일 목록의 모든 항목을 삭제합니다. 시스템 레지스트리에 저장된 최근 파일 목록이 모두 삭제됩니다.

프레임워크 대화상자

프레임워크 대화상자는 사용 가능한 프레임워크(Framework)를 선택하여 현재 프로젝트로 가져올 수 있도록 합니다.

Frameworks list

프레임워크 목록에는 등록된 프레임워크들의 아이콘과 함께 나타납니다. 여기서 가져오기를 원하는 프레임워크를 선택합니다.

Large Icon/Small Icon Button

프레임워크 목록에 나타나는 항목들의 아이콘을 크게 하거나 작게 나타낼 수 있습니다. 만약 프레임워크의 이름이 모두 보이지 않아 불편하다면 작은 아이콘으로 나타나도록 버튼을 선택합니다.

Description

프레임워크 목록에 사용자가 항목을 선택하면 해당 프레임워크에 대한 간략한 설명이 나타나는 영역입니다.

페이지 설정 대화상자

인쇄할 페이지에 어떤 다이어그램을 정보를 어떻게 표시할 것인지, 용지의 출력 방향이나 여백, 테두리 등을 조정할 수 있습니다.

Option

출력할 페이지에 나타날 몇가지 다이어그램의 정보에 관한 내용을 설정합니다.

  • On Header : 다이어그램에 관한 정보를 페이지의 머릿 부분에 출력합니다. 
  • On Footer : 다이어그램에 관한 정보를 페이지의 바닥 부분에 출력합니다. 
  • None : 다이어그램에 관한 정보를 출력하지 않습니다.
  • Diagram Name : 다이어그램의 이름을 출력합니다.
  • Project Title : 다이어그램이 포함된 프로젝트의 이름을 출력합니다.
  • Page Number : 출력될 페이지의 번호 출력합니다.
  • Diagram Kind : 다이어그램의 종류를 출력합니다.
  • Date/Time : 현재 날짜와 시간 출력합니다.

Margin

출력할 페이지의 위, 아래, 왼쪽, 아래쪽의 여백을 각각 밀리미터(mm) 단위로 조정합니다. 

Page orientation

페이지를 세로로 출력할 것인지 가로로 출력할 것인지를 결정합니다.

Border

출력할 페이지에 테두리를 어떻게 출력할 것인가를 조정합니다. 테두리가 나타날 위치 상, 하, 좌, 우를 선택하고 테두리 선의 굵기를 결정합니다.

인쇄 대화상자

사용자가 다이어그램의 인쇄를 원할 경우 나타납니다. 인쇄에 관련된 여러 가지 사항들을 사용자가 선택하고 결정할 수 있습니다.

Printer

프린터에 관련된 내용들을 설정합니다.

  • Name : 현재 설치된 프린터들 중 인쇄할 프린터를 선택합니다. 
  • Status : 선택된 프린터의 상태를 표시합니다.
  • Location : 선택된 프린터의 위치를 표시합니다.
  • Page Setup : 페이지 설정 대화상자를 나타나게 합니다.

Size

인쇄할 다이어그램의 크기를 설정합니다.

  • Diagram as is : 다이어그램 원래 크기 그대로를 인쇄합니다. 다이어그램의 크기가 용지를 벗어나는 경우 여러 장에 인쇄될 수 있습니다.
  • Fit to page : 다이어그램을 1페이지에 맞추어서 인쇄합니다. 다이어그램이 큰 경우 여러 장에 인쇄되는 것을 막을 수 있습니다.
  • Tile : 다이어그램의 크기를 원하는 크기만큼 확대하여 타일 형태로 여러 장에 나누어서 인쇄합니다. 인쇄할 용지의 수를 가로 장수, 세로 장수를 지정할 수 있습니다.(예: 가로 3장, 세로 4장 = 총 12장)

Print Range

인쇄할 다이어그램의 범위를 설정합니다.

  • Current diagram : 현재 활성 다이어그램만을 인쇄합니다.
  • All diagrams : 현재 프로젝트에 포함되어 있는 모든 다이어그램들을 인쇄합니다.
  • Selected diagrams : 아래에 나타나는 다이어그램의 목록 중에서 선택된 것 만을 인쇄합니다. [Select All] 버튼은 모든 다이어그램을 체크하고 [Deselect All] 버튼은 모든 다이어그램을 선택 해제합니다.

Preview

찾기 대화상자 

찾기 대화상자는 사용자로 하여금 원하는 요소를 쉽고 빠르게 찾을 수 있도록 도와줍니다.

Text to Find

찾을 내용 부분에는 찾고자 하는 요소가 포함하고 있는 문자열 중 일부를 입력합니다. 기존에 입력했던 내용들 중에서 선택할 수도 있습니다.

Option-Element Type

옵션 그룹의 요소 타입 부분에는 검색할 요소들의 범위를 지정합니다. 'All', Model, Subsystem, Package, Class, Interface, Enumeration, Signal, Exception, Component, Node, Instance, UseCase, Actor 중에서 한가지 범위를 선택할 수 있습니다.

Option-Case Sensitive

요소를 검색할 때 대문자와 소문자를 구분할 것인지를 지정합니다.

모델 검사하기 대화상자 

모델 검사하기 대화상자는 사용자가 모델 요소들이 잘 정의되어 있는 가를 검사하기 위해서 사용됩니다.

Verification Rule

모델 검사를 시작하게 되면 현재 적용하고 있는 검사 규칙이 나타나는 영역입니다.

Verifying Element

현재 검사중인 요소의 이름이 나타납니다.

Progress

검사가 얼마나 진행되었는가를 시각적으로 보여줍니다.

Failed

검사에서 탈락한 요소가 현재 몇개인가를 나타냅니다.

옵션 대화상자 

옵션 대화상자는 StarUML™의 여러 가지 환경 설정에 관한 옵션 항목들을 열거하고 편집할 수 있도록 합니다.

Option category

옵션에 대한 분류들이 나타납니다. 가장 큰 상위 분류로 '환경'이 있으며 그 하위로 '일반', '다이어그램', '일반 뷰', '특정 뷰'의 하위 분류들이 나타납니다. 여기에 나타나는 옵션 분류는 StarUML™에서 작동되는 모듈 등에 의해 새로운 것들이 더 추가될 수도 있습니다.

Option item

옵션 분류 목록에서 선택된 분류에 포함되는 옵션 항목들이 나타나고 그 값을 편집할 수 있습니다.

Description

선택된 옵션 분류 혹은 항목에 대한 간략한 설명이 나타납니다.

Reset to default values

선택된 옵션 항목의 값을 기본으로 설정된 값으로 되돌립니다.

Revert to last value

선택된 옵션 항목의 값이 변경된 경우, 값을 변경하기 전으로 되돌립니다.

스테레오타입 선택 대화상자 

사용자가 선택된 요소에 대해서 스테레오타입을 지정하고자 하는 경우에 나타납니다. 스테레오타입을 직접 입력하거나 목록 중에서 선택할 수 있습니다.

Stereotypes

사용자가 스테레오타입을 직접 입력할 수 있습니다. 스테레오타입이 등록된 것이 있으면 스레테오타입 목록에 곧 바로 나타납니다.

스테레오타입 목록

프로젝트에 사용되고 있는 UML 프로파일들에 정의되어 있는 스테레오타입들이 나타납니다. 스테레오타입의 이름과 그것이 포함된 프로파일의 이름이 함께 나타납니다. 사용자는 이 중에서 하나를 선택할 수 있습니다.

Icon preview

선택된 스테레오타입에 아이콘이 연결되어 있는 경우 아이콘 그림이 나타납니다. 없으면 나타나지 않습니다.

Description

선택된 스테레오타입에 대한 설명이 나타납니다.

요소 선택 대화상자

요소 선택 대화상자는 프로젝트 요소들의 계층 구조에서 사용자가 한 요소를 선택할 수 있도록 합니다. 요소 선택 대화상자는 프로퍼티 편집기 혹은 컬렉션 편집기 등에서 특정 프로퍼티에 요소를 할당하기를 원할 때 나타납니다. 요소 선택 대화상자는 요소 목록 대화상자와는 달리 요소들이 계층적인 구조로 나열됩니다.

대화상자 제목 

대화상자 제목은 어떤 종류의 요소를 선택해야 하는가에 따라서 달라집니다. 속성(Attribute)의 타입을 지정하는 경우, 객체의 타입 즉, 분류자(Classifier)를 지정하는 경우에 따라 적합한 제목이 나타납니다.

요소 목록 

요소 목록에는 선택할 수 있는 요소들의 나타납니다. 예를 들어 상태머신(StateMachine)을 선택해야 하는 경우에는 상태머신만이 목록에 나타납니다.

Data Types

데이터타입 목록에는 기본으로 사용할 수 있는 데이터타입들이 나타납니다. 여기에 나타나는 데이터타입은 현재 프로젝트에서 사용하고 있는 UML 프로파일에 정의되어 있는 것입니다. 이 목록은 데이터타입을 지정하기 위한 상황이 아닌 경우에는 나타나지 않을 수 있습니다.

Do not specify　

아무것도 지정하지 않으려면 이 곳을 체크합니다. 이것의 정확한 의미는 Null 값을 할당하는 것입니다.

선택 요소 표시

대화상자의 아랫 부분에 현재 선택된 요소의 완전 경로명이 나타납니다. 이것으로 현재 어떤 요소가 선택되어 있는지를 확인할 수 있습니다.

요소 목록 대화상자   

요소 목록 대화상자는 사용자가 목록 중의 한 요소를 선택할 수 있도록 합니다. 요소 목록 대화상자는 프로퍼티 편집기 혹은 컬렉션 편집기 등에서 특정 프로퍼티에 요소를 할당하기를 원할 때 나타납니다. 요소 목록 대화상자는 요소 선택 대화상자와는 달리 요소들이 1차원적인 리스트로 나열됩니다.

대화상자 제목

대화상자 제목은 어떤 종류의 요소를 선택해야 하는가에 따라서 달라집니다. 하나의 상태머신(StateMachine)을 선택하는 경우 혹은 노드(Node)에 컴포넌트(Component)를 배치하는 경우 등은 각각 상황에 맞는 제목으로 나타납니다.

요소 목록 

요소 목록에는 선택할 수 있는 요소들의 나타납니다. 예를 들어 상태머신(StateMachine)을 선택해야 하는 경우에는 상태머신만이 목록에 나타납니다.

Do not specify　

아무것도 지정하지 않으려면 이 곳을 체크합니다. 이것의 정확한 의미는 Null 값을 할당하는 것입니다.

모델 걸러내기 대화상자

편집 영역에서는 역할의 스레테오타입, 가시성, 이름 그리고 타입을 구문(syntax)에 따라 입력할 수 있습니다.

구문 

<< stereotype >> visibility name : type

  • << stereotype >> : 스테레오타입의 이름을 입력합니다.  이 부분은 생략 가능합니다. 
  • visibility : 역할의 가시성에 해당하는 문자('+': public, '#': protected, '-': private, '~': package)를 입력합니다. 이 부분은 생략 가능합니다.
  • name : 역할의 이름을 입력합니다.
  • : type : 역할의 타입 이름을 입력합니다. 반드시 현재 프로젝트 내에 정의된 분류자(Classifier) 중 하나이어야 합니다. 이 부분은 생략가능합니다.

새로운 Class 요소 생성 버튼 

ClassifierRole 요소가 속한 Collaboration의 상위 Namespace에 새로운 Class 요소를 생성하고, ClassifierRole의 Base 속성이 생성된 Class 요소를 참조하도록 합니다.

요소 연결 버튼 

요소 연결 버튼은 메시지 혹은 자극의 종류에 따라 특정 요소를 연결할 수 있도록 합니다. 메시지 혹은 자극이 호출 액션(CallAction)을 가지는 경우에는 상대편 객체의 연산(Operation)들 중 하나를 선택할 수 있고, 생성 액션(CreateAction)인 경우는 분류자(Classifier)를 연결할 수 있습니다. 그리고 전달 액션(SendAction)인 경우에는 시그널(Signal) 요소를 연결할 수 있습니다.

가시성(Visibility) 버튼 

메시지 혹은 자극의 가시성을 Public, Protected, Private, 그리고 Package 중에서 선택할 수 있습니다.

편집 영역   

편집 영역에서는 메시지 혹은 자극의 이름, 가시성, 그리고 스테레오타입을 구문(syntax)에 따라 입력할 수 있습니다.

구문 

<< stereotype >> *[iteration] [condition] return := messagename ( arguments )

  • << stereotype >> : 스테레오타입의 이름을 입력합니다.  이 부분은 생략 가능합니다.  
  • *[iteration] : 메시지 혹은 자극의 반복식을 입력합니다. 예를 들면 "*[i=1..100]" 과 같이 입력할 수 있습니다. 이 부분은 생략 가능합니다.
  • [condition] : 메시지 혹은 자극이 수행될 조건을 입력합니다. 이 부분은 생략 가능합니다. 
  • return := : 메시지 혹은 자극이 수행된 결과를 담는 식을 표현합니다. 이 부분은 생략 가능합니다.
  • messagename : 메시지 혹은 자극의 이름을 입력합니다. 
  • ( arguments ) : 메시지 혹은 자극으로 전달될 인자들의 식을 표현합니다. 이 부분은 생략 가능합니다.

새로운 Operation 요소 생성 버튼
